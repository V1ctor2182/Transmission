# Hero 功能 + Loop v2 设计(分层 · 双模式)

> 给 loop 的新输入。核心:**不是每屏都一样。** Hero 功能走「放大模式」(加体验包装、按流程+动效验收、人工把关);其余走「清理模式」(去 AI 味、静态验收、自动落库,即当前 T1–T7 那套)。

---

## 🎚️ 分层(loop 每轮先判这一屏属于哪档)

| 档 | 是什么 | loop 怎么做 | 验收 | 落库 |
|---|---|---|---|---|
| **Hero** | 销售重点要卖的 3 个(见下) | **放大模式**:多轮(概念→动效→打磨→demo 验证),投入不设上限 | **多帧序列截图 + 黄金路径端到端 + delight rubric** | **提到分支 + 暂停等人工点头**(不自动 merge) |
| **Standard** | 营销/邮件、情报、客户池 | 清理模式:一轮去 AI 味 | 单帧 critic 多数 | 自动落库 |
| **Utility** | 设置/纯列表/弹窗 | 最快过:机械去 emoji/统一图标 | 机检断言 | 自动落库 |

**Hero 的闸门 = 在四段基础上加这三样(静态 PNG 判不了 wow):**
1. **序列截图**:同一交互截 t=0 / t≈1.5s / t≈3.5s 三帧,让 critic 看**动效的弧线**而不是一帧。
2. **黄金路径脚本**:整条 demo 流端到端跑一遍(无死胡同、无报错、每步该有的反馈都在)。
3. **delight rubric**(另写一套):judge「这会不会让买家眼前一亮 / wow 是不是挣来的 / time-to-first-wow 够快」,而不只是「有没有 AI 味」。
4. **Hero 不自动落库** —— 强主观高风险,loop 产出带动效预览后**暂停等你拍板**。

---

## 🚫 贯穿所有 Hero 的一条红线(最重要)
**「活」必须是挣来的(真数据、真意义),绝不能是假的。** 包装恰恰是 AI 味回潮的老家(我们 T4 才删掉一个每 3.5s 乱跳的假计数)。
- ✅ 好 wow:`journey-preview` 里「分析 = 指挥台被你真实市场填满」、买家带匹配理由流入。
- ❌ 坏 wow:假百分比进度条、为动而动的脉冲、空转自增的计数器、满屏「实时」徽标。
- 判据:每个动/每个「实时」,问一句**它对应哪个真实事件**?答不上来就是装饰,删。

---

## 🌟 Hero 1 — AI 分析首启(注册即分析)
- **卖点(一句)**:30 秒,从你的网址看见你的全球买家 —— 零表单、零等待感,它**当场就懂你的市场**。
- **Demo money-shot**:输完官网,指挥台**当着客户的面自我拼装**(地图一个区域一个区域点亮 → KPI 从 0 滚上来 → 买家一行行「刚发现」流入),收束成一句「为[万仟糕饼]锁定 2.8M 商机 · 4 个火热区域 · 12 条高匹配」。
- **要包装的体验**:拼装编排的**节奏**(先地图后数据后买家,有起承转合)、诚实状态文案(文字在变不是假 %)、收束「settle」的满足感、可随时「直接进入」。已出预览 `journey-preview.html`。
- **AI 味红线**:点亮/计数必须对应**真识别到的数据**;无假 %;骨架 loading 用真实形状。
- **体验验收**:序列截图看出**渐进揭示**;golden path `输网址→拼装→ready` 不断;盲评「buyer 会不会觉得『它已经懂我的市场了』」;time-to-first-wow < 2s(地图第一个点亮要快)。

## 🌟 Hero 2 — 找客户 / ICP Agent
- **卖点(一句)**:AI 自动从全球数据源,找到你**自己够不着**的精准买家 —— 还打好分、给得出**为什么匹配**。这是「不公平信息优势」的内核。
- **Demo money-shot**:点「开始找客户」→ 买家带**匹配理由**一行行流入(来源:海关数据/LinkedIn;理由:采购品类/规模/频率),点 enrich → **锁定的联系方式解锁**(邮箱/电话)。「它找到 12 个我这辈子找不到的买家,还带邮箱」。
- **要包装的体验**:真·流式发现(motivated,不是假滚动)、**每个买家可见的打分理由**(透明=信任)、enrich 的「锁→解锁」揭示(价值兑现的瞬间)、地区/品类筛选 chip 的掌控感。
- **AI 味红线**:**不准**用假乱跳计数(已删);流入的必须是真结果;分数必须有**可见的理由**,不能只甩一个数字。
- **体验验收**:序列截图看出买家**带理由地到达**;enrich 解锁流程跑通;golden path `开始找→结果→enrich→加入客户池`;盲评「这是在找真买家,不是在演」。

## 🌟 Hero 3 — 实时商机 / 建联
- **卖点(一句)**:全球采购需求**实时可见**,看到热点**一键就能建联对话** —— 从信号到对话只差一步,先一步行动。
- **Demo money-shot**:活的热力图(热点在firing)→ 点一个火热区域 → 该区域买家 → **一键建联** → WhatsApp 对话打开,带 **AI 话术建议**(已清干净)。「从看见热点到开口聊,一次点击」。
- **要包装的体验**:地图作为**活的中心**(sonar ping = 真信号)、热点→下钻→建联的连贯流、WhatsApp AI 话术(真实、可用)、「建联次数」作为可感的货币。
- **AI 味红线**:ping/「实时」只在**真有实时数据**处用;不到处脉冲;话术 chip 不带装饰 emoji(已做)。
- **体验验收**:golden path `热力图→点热点→买家→一键建联→对话` 端到端;动效 motivated(每个 ping 对应一条信号);盲评「看得见、聊得上」的即时感。

---

## 🔗 别忘了:Demo 是一条流,不是一堆屏
销售演示是一条叙事路径。除了逐个 Hero,**单独留一个「黄金 demo 路径」任务**:`输网址 → 分析拼装 → 进工作台 → 找客户找到买家 → 建联开口 →(可选)AI 邮件`,打磨**整条流的节奏**(哪步给惊喜、哪步要快、零死胡同)。这条流顺不顺,比任何单屏都更决定成交。

## ⚙️ 怎么把这套接进 loop
1. loop 每轮先读本文件判档(Hero / Standard / Utility)。
2. Hero 轮:`verify.mjs` 加**序列截图**(3 帧)+ **golden-path 脚本**;critic 换 **delight rubric**;落库改成 `git commit 到 feat/hero-xxx 分支 + 暂停`,等你 review 预览后再 merge。
3. Standard/Utility 轮:沿用现有四段闸门 + 自动落库。
4. 红线(挣来的「活」)写进每个 Hero 轮的 critic prompt,双向防 AI 味回潮。
