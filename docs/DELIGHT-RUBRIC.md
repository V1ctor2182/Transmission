# Delight Rubric — Hero 功能专用评审标准

> 给 Hero 轮的 critic 用(替代 `REVIEW-RUBRIC.md` 的纯 anti-slop 评审)。
> critic 看的是**序列截图**(t0 / t1 / t2 三帧,动效的弧线),不是一帧。fresh eyes、不知道改了啥。

## 任务
判断这个 Hero 交互**会不会让一个潜在买家眼前一亮**,且 wow 是**挣来的**(真数据/真意义),不是装饰。给二元裁决 `KEEP` / `REVERT`。

## 三帧序列怎么读
- t0(刚进/刚点)→ t1(进行中)→ t2(收束)。看**渐进揭示**:有没有起承转合?信息是不是一层层长出来的?最后有没有一个让人「哦!」的 settle?
- 反面:三帧几乎一样(没动效)、或者一上来全都有了(没有揭示的爽感)、或者动了但没意义(为动而动)。

## 评分维度(每项 A–F)
1. **wow / 第一印象**:t2 那帧会不会让人想截图发群?有没有一个清晰的「money-shot」?
2. **wow 是不是挣来的(最重要)**:每个动、每个数字、每个「实时」,**对应一个真实事件吗**?(地图点亮=真识别到区域;买家流入=真结果;count-up=真数值)。**只要发现一处假的(假 %、空转计数、为动而动的脉冲)→ 直接 REVERT。**
3. **time-to-first-wow**:第一个惊艳点来得够快吗?(t1 之前就该有东西亮起来,别让人干等)。
4. **叙事节奏**:揭示顺序合理吗?(该先给概览还是先给细节?该快的快、该停的停?)
5. **不掉链子**:三帧里有没有半成品/错位/卡住/死胡同的迹象?

## 硬否决(任一 → REVERT)
- 发现**假的「活」**(假进度条/假 %/空转自增计数/无意义脉冲)—— 这是我们一直在删的 AI 味,Hero 包装里更不许回潮。
- 三帧无变化(声称有动效却没动)或动效卡顿/半成品。
- 比改之前更像「demo 剧场」(炫技 > 信息)。
- 文字看不清 / 渲染崩坏。

## 输出(严格 JSON,最后一行)
```
__DELIGHT__ {"verdict":"KEEP|REVERT","wow":"A-F","earned":"A-F","ttfw":"A-F","reasons":["指出哪一帧的哪个像素级证据"],"fake_liveliness":["发现的假『活』,或空"],"money_shot":"t2 那帧最值得截图发群的是什么,或 none"}
```
- `verdict=REVERT`:命中任一硬否决,或 `earned ≤ C`,或 `fake_liveliness` 非空。
- 不确定 → REVERT(默认怀疑)。
